# Security Policy

## Supported Versions

Only the latest stable release receives security fixes.

| Version      | Supported |
| ------------ | --------- |
| 1.x (latest) | Yes       |
| < 1.0        | No        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

To report a vulnerability, use one of the following private channels:

- **GitHub:** [Report a vulnerability](https://github.com/mateusvillain/lets-ui/security/advisories/new) via GitHub's private disclosure form
- **Email:** [mateus@mateusvillain.com](mailto:mateus@mateusvillain.com)

Include as much detail as possible:

- Description of the vulnerability and its potential impact
- Steps to reproduce or proof-of-concept
- Affected versions
- Any suggested fix, if you have one

## Response Process

1. You will receive an acknowledgment within **48 hours**.
2. The maintainer will investigate and confirm the issue within **7 days**.
3. A fix will be prepared and released as soon as possible, with a coordinated disclosure date agreed upon with the reporter.
4. A [GitHub Security Advisory](https://github.com/mateusvillain/lets-ui/security/advisories) will be published after the fix is released.

## Scope

This policy applies to vulnerabilities in the published packages:

- `@lets-ui/tokens`
- `@lets-ui/styles`
- `@lets-ui/components`

Vulnerabilities in development-only dependencies (e.g. Storybook, build tooling) are lower priority but still welcome.
